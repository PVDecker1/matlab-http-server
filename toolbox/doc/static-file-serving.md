# Static File Serving

`matlab-http-server` includes a built-in mechanism to serve static assets like HTML, CSS, JavaScript, and images directly from your MATLAB environment.

## Basic Usage

Use the `serveStatic` method to register a directory for static file serving.

```matlab
server = MatlabHttpServer(8080);
server.serveStatic("public/");
server.start();
```

By default, files are served from the root URL path (`/`).

## Same-Origin Frontend + API

One useful pattern is to serve your frontend assets and API routes from the same MATLAB server.

```matlab
server = MatlabHttpServer(8080);
server.register(MyController()); % handles /api/...
server.serveStatic("public/");
server.start();
```

This keeps the browser app and the API on the same origin, which simplifies local tools and internal dashboards.

## URL Prefixes

You can serve static files under a specific URL prefix using the `UrlPrefix` option.

```matlab
server = MatlabHttpServer(8080);
server.serveStatic("doc/", UrlPrefix="/documentation/");
server.start();
```

In this example, a file at `doc/index.html` on disk would be accessible at `http://localhost:8080/documentation/index.html`.

## Interaction with API Router

Static handlers are checked **before** the API router. If a static handler successfully serves a file matching the request path, the request is completed. If no file matches, the request "falls through" to the registered `ApiController` routes.

```matlab
server = MatlabHttpServer(8080);

% Register API controllers
server.register(MyController()); % handles /api/...

% Register static handler
server.serveStatic("public/");   % serves everything else; falls through if no file matches

server.start();
```

## Features and Behavior

### Directory Resolution
If a requested path corresponds to a directory on disk, the server automatically looks for an `index.html` file within that directory.
- Request for `/` -> serves `public/index.html`
- Request for `/about/` -> serves `public/about/index.html`

### Path Traversal Protection
The server uses canonical path resolution to ensure that requested files reside within the specified root directory. Requests attempting to use `..` segments to access files outside the root will receive a `403 Forbidden` response.

### MIME Type Detection
The server automatically sets the `Content-Type` header based on the file extension. Supported extensions include:

| Extension | MIME Type |
| :--- | :--- |
| `.html` | `text/html; charset=utf-8` |
| `.css` | `text/css` |
| `.js` | `application/javascript` |
| `.json` | `application/json` |
| `.xml` | `application/xml` |
| `.svg` | `image/svg+xml` |
| `.png` | `image/png` |
| `.jpg` / `.jpeg` | `image/jpeg` |
| `.ico` | `image/x-icon` |
| `.woff` | `font/woff` |
| `.woff2` | `font/woff2` |
| `.ttf` | `font/ttf` |
| `.webp` | `image/webp` |

Files with unknown extensions are served as `application/octet-stream`.

### Binary Content Stability
Binary assets (images, fonts, etc.) are served using raw bytes via `HttpResponse.sendBytes`. This avoids the UTF-8 encoding applied by the standard `send()` method, which can corrupt binary data.

## Hugo Site Example

You can easily serve a static site generated by tools like [Hugo](https://gohugo.io/).

1. Generate your site: `hugo` (usually outputs to a `public/` folder).
2. Serve it from MATLAB:

```matlab
server = MatlabHttpServer(8080);
server.serveStatic("public/");
server.start();
```

## Known Limitations

- **No Caching Headers**: The server does not currently send `ETag` or `Cache-Control` headers.
- **No Range Requests**: Partial content requests (useful for large video files) are not supported.
- **No Directory Listing**: Requesting a directory without an `index.html` results in a fallthrough (usually a 404 from the router).
- **No Compression**: Files are served exactly as they are on disk without GZIP or Brotli compression.

## Production Recommendation

For high-traffic production use cases, it is recommended to use a dedicated web server like [Caddy](https://caddyserver.com/) or Nginx to serve static assets and act as a reverse proxy for your MATLAB API.

```
Caddy (:443) 
  -> handle_path /api/* { reverse_proxy localhost:8080 }
  -> file_server { root /var/www/html }
```
